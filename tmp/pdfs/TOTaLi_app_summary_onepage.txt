TOTaLi App Summary (Repo Evidence)

What it is:
- TOTaLi is a local spatial drafting pipeline for point cloud survey data.
- The code separates probabilistic AI classification from deterministic
  geometry generation and human PLS certification.

Who it is for:
- Licensed Professional Surveyors (PLS) and survey drafting teams that
  need auditable LiDAR-to-CAD outputs.

What it does:
- Runs a 5-phase CLI pipeline: geodetic, segment, extract, shield, lint.
- Can run a single phase or all phases.
- Validates CRS/epoch/units at ingest and standardizes coordinates.
- Classifies points with ONNX when available; falls back to deterministic
  rule-based classification.
- Extracts deterministic DTM/TIN, breaklines, contours, and planimetric
  features.
- Heals/quarantines geometry, writes DRAFT-only CAD layers, and exports
  DXF plus entity manifest.
- Generates lint report and review worksheet; auto-promote is disabled.
- Writes JSONL audit logs with SHA-256 hash chaining.

How it works (architecture):
- Entry: totali.main loads YAML config and initializes
  PipelineOrchestrator and AuditLogger.
- Data flow: LAS/LAZ -> GeodeticGatekeeper -> PointCloudClassifier ->
  DeterministicExtractor -> CADShield -> SurveyorLinter.
- State flow: each phase returns PhaseResult; outputs are passed forward
  in a shared context dictionary.
- Artifacts: gated LAS, extraction/lint reports, draft DXF, entity
  manifest, and hash-chained audit log.
- External runtime cloud services: Not found in repo.

How to run (minimal):
1) Install dependencies: pip install -r requirements.txt
2) Run full pipeline:
   python -m totali.main --input path/to/pointcloud.las
   --config config/pipeline.yaml
3) Check artifacts in output/ and audit logs in audit_logs/.
4) Optional: run one phase with --phase geodetic|segment|extract|shield|lint
